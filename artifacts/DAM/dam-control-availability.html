Copy

These messages come from three families — each a template populated per feature, so there's no need to write a separate message for every tab. Most are family A: one template, where you swap the feature name and its supported formats.

A · Format relevance

Tab → render title + body in the panel. In-panel control → render the body only, as a tooltip. Same source string either way.

Title {feature} not available Body {feature} is only available for {formats}. render `${feature} is only available for ${joinOr(formats)}.` // joinOr(["JPEG","PNG","WebP"]) → "JPEG, PNG, or WebP"

The renderer is written once; each feature supplies its own row:

Feature (tab){feature}{formats} — supported
AdjustmentsImage adjustmentsJPEG, PNG, WebP
SimilarVisual similarityJPEG, PNG, WebP
AI TagsAI taggingJPEG, PNG, WebP
OCRText recognitionJPEG, PNG, WebP, PDF

List supported formats when that set is short (the common case). Flip to the negative — Not available for {fileType} files. — only when the supported set is long and the excluded set is short. Adding a feature means adding a row, not new code.

B · Rights

You don't have rights to {action}. [Request access]

Stays active and routes — not disabled. e.g. You don't have rights to download the original. [Request access]

D · AI result states — async: AI Tags, Similar

These tabs are async and the results are AI-generated, so every state carries the AI treatment — two of three: gradient/shimmer, sparkle icon, AI badge. Lifecycle:

StateAI TagsSimilar
Generating first run — model computingGenerating tags…Finding similar assets…
Loading fetching cached resultsLoading tags…Loading similar assets…
Empty ran, nothing foundNo tags generated for this asset.No visually similar assets found in this library.
ErrorCouldn't generate tags. Retry.Couldn't load similar assets. Retry.

Generating is slower than loading (the model is running), so use an AI shimmer skeleton — not a bare spinner — and add a "This can take a moment." subline if it may exceed a couple of seconds. Loading is a brief skeleton. Format-unavailable uses family A, not this lifecycle.

Mechanics

Disabled = opacity:0.5; cursor:not-allowed; color:var(--text-subtle) (Scale .btn:disabled), aria-disabled="true", and keep focusable so the tooltip is reachable by keyboard. View/nav tabs use standard tab semantics — never aria-disabled; the panel content is the announced state.

The line

AI features and transformations/adjustments run on raster image formats only. Every other format gets asset management + whatever preview its type supports.

This matrix says which formats unlock each capability; the rules above say how to present a blocked one.

Note

  • Vector AI is deferred, not impossible. We can generate the raster previews vector AI tagging/similarity would need, but the token cost isn't justified yet. Revisit when that changes — the only change required is flipping the vector cells on the AI Tags and Visual similarity rows.
  • OCR is the one AI feature that extends past raster — it runs on PDFs (scanned/text), so it's ✓ for PDF.
  • "Raster" is the gate; the adjustment editor's real supported set is narrower — JPEG, PNG, WebP. Drive the panel message from that actual list, not the broad raster group: TIFF / HEIC / RAW may preview but not adjust.
  • Audio uses the same player as video — preview/playback ✓. AI and transforms still don't apply.
  • Bulk download = lowest common quality across the selection — a tier unavailable for any one asset is unavailable for the set.

FIFA

Rule: tab → always active, message in panel — identical to the lightbox. Copy lists the supported formats (more actionable than naming the excluded type). Per-control disable + tooltip is reserved for individual options inside an applicable panel (a no-upscale size, a rights-locked quality tier), not for a whole feature that doesn't apply.
