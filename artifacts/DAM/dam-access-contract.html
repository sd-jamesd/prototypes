Access contract — public links (CDN)

Publish is not a rail stop. It is a section inside Access. Access answers one question — who can reach this asset — across two sections: People & groups (internal principals) and Public links (the anonymous principal). Rights answers a different question and keeps its own tab. Built entirely from the kit; no net-new atoms.

Publish — structural action, own control + confirm

Publish is not a toggle. The cross-cutting rule says structural / immediate actions get their own control and a confirm, and bypass the global Save — "don't let an immediate action masquerade as a staged edit". A toggle is exactly that masquerade: it reads as a setting that stages. Minting an unsigned, externally-addressable URL is the most consequential immediate action in the panel. It gets a button and a confirm.

  • The confirm names the exclusions. The panel's disabled rows say what can't publish; the confirm says what will. Both, or the user learns the constraint after the fact.
  • Auto-revoke is stated at mint time. The user is told the link has an end date before they paste it into a partner's CMS — not when it 404s.
  • Bulk publish gets the same confirm, with the intersection stated: "9 of 12 will publish · 3 blocked by rights · 4 presets common to all 9."

  • Copy as <img> and UTM are per-rendition. This is where the export need lands honestly — one link, one format, one purpose. Not a "copy all" blob.

  • Delivery stats has a slot, not an item. View/download counts per URL are in the CDN scope but not built. It goes here when it exists — a dead menu item is worse than an absent one.

Which templates publish? — not all of them

Publishing an asset does not mint a URL for every template. It mints one for every shared template an admin has promoted to the public delivery channel, minus the ones the asset's rights forbid. Most templates are download-only — print, working files, masters. The deliverable set is small: three to five.

The confirm — flipping channel is a tenant-wide publish event

Make "Print 300dpi" publicly deliverable?
This preset will become a public CDN output on every published asset whose rights permit print use.
New public URLs38,412 across 41,006 published assets
Excluded by rights2,594 — digital-use-only licences
ReversibleYes — unticking revokes and purges all 38,412
  • This is the hole. A checkbox in template settings looks like config. It is in fact the single highest-blast-radius action in the DAM — one tick exposes a new public rendition of every published asset in the tenant.
  • It needs a blast-radius confirm stating the count, the exclusions, and the reversal cost. Same pattern as the asset-level publish confirm, three orders of magnitude louder.
  • Untick is worse than tick. Revoking 38,412 URLs breaks every embed using them — partner websites, CMS templates, email footers. That confirm must be as loud, and it should probably require a reason for the audit log.
  • Default off. A new preset is download-only. Public delivery is an opt-in, not something you inherit by creating a crop.

Create a template — no, the channel doesn't go here

The blocker isn't the surface. It's the missing field underneath. A channel needs a scope to hang off, and the create modal has none. Right now "Design test" and "CK T1 Edited 1" sit in the same list as "Instagram Square" — the research called this exactly ("without governance, presets become junk fast") and it has already happened. Add a public-delivery checkbox to that modal today and anyone can name a crop "Design test" and mint a new public rendition of every published asset in the tenant.
  • Scope is the prerequisite. Personal vs Shared, with Shared as admin-only. That's already the governance model in the research — it was just never built.
  • Create ≠ promote. A new template has never produced a single derivative. Nobody has looked at its output. Committing 38,000 public URLs to an unreviewed crop is an unforced error. You create, you look, then an admin promotes.
  • So channel lives on edit, not create — reached from the template row's ⋯ in Saved templates, admin-only. Same surface, different act. No new admin section needed to ship this.

  • The real need is the transpose. One preset across forty assets — a dev building srcset, a producer building a brief. That's a library-level bulk action on a multi-select, not per-asset panel chrome.

Expiry — two clocks, not one

Rights expiry is a ceiling. Auto-revoke is a choice. They are not the same field and conflating them is how you build a liability. Rights expiry lives in Rights, is non-negotiable, and cascades to every link. Auto-revoke lives here, is optional, and can only ever shorten — the picker's max is the rights ceiling. Effective date = min(rights_expiry, auto_revoke). Same backend either way: revoke → edge purge.
FieldLives inBehaviour
Rights ceilingRightsRead-only here. Legal limit. Cannot be raised or bypassed from Access.
Auto-revokeAccess · Public linksOptional, empty by default. Hard-capped at the rights ceiling. Stages to the global Save — it isn't immediate, so it isn't a confirm action.
  • Why it earns a field: a licence can run to 2028 while the campaign ends in March. Without this you leave the asset publicly live for three years because nobody remembered to revoke. Stale public links are the single most common source of leaked assets in DAM.
  • What it is not: a share-link expiry. A CDN link exists to be embedded permanently — an expiring embed is a broken partner website. A time-boxed, gated link is a different object, and that's SCR-62. Don't build a third thing here that's neither.
  • Set at publish, changed in panel. The confirm carries the field so the decision lands at the moment of consequence; the panel carries it so it can be added later.

Open calls

  • The contract contradicts itself on the panel header. Foundations says panel header = font-size-3 · 16px, and bans 18px outright: "There is no 13px, 15px or 18px." The Panel contract tab says "18px / 600." I built 16. If the merge script's type-ladder assertion isn't catching this, it should be.
  • Two Primaries. Grant access and Publish both create data, and the tab allows one Primary. I made Publish the Primary — it's the only action that mints an artefact outside the system — and demoted Grant access to Secondary. That's a call, not a clause. Overrule it if you read it the other way.
  • "Embed" is in the rail's tool list. Foundations names edit-only tools as Adjustments · Template · AI · Versions · Embed, but the worked-example rail has no Embed stop. If Embed was the old name for this work, it's now dead — Access absorbed it, and the contract line needs deleting.
  • UTM tagging — held. Out of this phase. Slot reserved in the row ⋯ menu, below Copy as <img>. PRD 4.8.4.
  • Template scope does not exist yet. Everything here — channel, public delivery, governance — is blocked on adding Personal / Shared to templates. That's the ticket that comes first.
  • Purge SLA. Still open, still the thing to take to Mark. Bynder and Brandfolder both market real-time invalidation. ASDA's RFP will ask for a number.
