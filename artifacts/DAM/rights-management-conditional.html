Behaviour for Rights-Managed

Rights-Managed assets have strict contractual terms. Every dimension of usage must be defined — system blocks save until all required fields are completed.
Territory Required User must pick country-specific, region, or global. Save blocked otherwise.
Channels Required At least one channel must be selected.
Audience Required At least one audience must be selected.
Embargo Date Optional Set if asset is not yet live.
Expiry Date Required Rights-Managed assets must have an end of licence date.
Public sharing Optional Allowed, but share links must respect territory and channel.
Note on the model: Changing Licence Type does not wipe Territory / Channels / Audience selections — it changes which of them are required, optional, hidden, fixed, or constrained. The cascade is on the rules, not the data. The "This licence permits use:" divider exists so the dev / user reads everything below it as terms of the single licence above, not as five independent settings.
