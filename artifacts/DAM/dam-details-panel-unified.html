Reading the mock

  • Editable fields (Title, Description) — inputs in edit mode, static text in view mode. The set is identical; only the affordance changes. This is disable/show, don't hide applied to the view/edit split.
  • Read-only facts render identically in both modes — they were never editable, so there's no state to toggle. The editor's job for these is view-only, same as the lightbox.
  • Activity (new): read-only audit trail of every action on the asset — the UI for the audit-history panel already required by the rights spec. Orthogonal to the other tabs: they own live state and emit typed events (actor · action · target · before→after · timestamp · source); Activity reads them. An access or rights change is one line item. Distinct from Versions (content lineage): a version upload appears in both — Versions owns it, Activity references it. Append-only, source-tagged (human/AI/automation), filterable by category + actor, role-scoped content. Inherently read-only, so it renders identically in edit and lightbox.
