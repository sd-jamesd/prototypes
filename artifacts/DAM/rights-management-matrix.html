Rights Management — Outcome Matrix

Three matrices, one per asset state. Each cell shows what happens when a user attempts to download an asset of a given licence type. Hover any cell for the reason and the rule that produced it. Use as a single-page reference during build and QA.

User columns reference the canonical QA test plan personas (U03, U04, U06, U07, U09) so this matrix can be cross-checked against the download QA tester. The Public Share Link column has no QA-tester equivalent — it represents the unauthenticated public-share case.

Outcomes: Cleared — Full-res delivered Watermarked — Working copy only Blocked — No file, no recovery Request Access — Recoverable via approval Privileged Override — Admin / owner bypass Hover any cell for full reason & rule trigger

Rule precedence

The rights engine evaluates rules in this order. The first rule that fires determines the outcome — subsequent rules don't run. This is why expiry always short-circuits licence-specific logic, and why Internal Only assets never reach the territory check.

0
Privileged-role bypass (Pre-Rule)
Before any rights check fires, the system evaluates whether the user holds an override role. Superadmin (U01, JWT claim) bypasses unconditionally. DAM Manager bypasses for all tenant assets with audit logging. Asset Owner bypasses for own assets, with an acknowledgement modal for expired downloads. Override actions are logged with the override reason.
Role match → override
No role match → proceed to Rule 1
1
Asset state check
If asset is Expired → hard block all users (no Request Access). If Embargoed → block external users, allow watermarked for internals (production needs it).
Short-circuits all other rules
2
Licence type gate
If Internal Only → block all external users immediately (no Request Access path exists for this licence type). If Owned → skip rights checks entirely, proceed to delivery.
Owned → cleared
Internal Only + external → blocked
3
Audience match
User's audience (Internal / Agency / Press / Editorial / Public) must intersect with the licence's allowed audiences. Audience is a property of the user, not of their role. A "Designer" can be Internal or Agency; the audience determines rights eligibility, not the role. Mismatch with no recovery path = hard block. Mismatch where user could plausibly need access = Request Access.
Match → continue
Mismatch + recoverable → request
Mismatch + not recoverable → blocked
4
Territory match
User's home territory must intersect with the licence's allowed territories. Internal users get a watermarked working copy on territory mismatch (graceful degrade). External users are blocked.
Internal mismatch → watermark
External mismatch → blocked
5
Role-inferred intent (channel match)
User's likely channel of use is inferred from role (not audience): Marketing role → commercial intent; Editorial role → editorial intent; Design role → reference/production intent. Editorial Only assets block users whose role implies commercial use, even when audience passes. This is where audience and role do distinct work.
Marketing role + Editorial asset → blocked
Editorial role + Editorial asset → cleared
Design role + Editorial asset → watermark (reference only)
6
Tenant policy applied
Tenant configures the global enforcement mode: Strict (any mismatch = hard block, no watermark fallback) or Soft (watermark fallback enabled for internal users). FIFA = Strict. Lighter clients = Soft. This matrix shows Soft (the more permissive default).
Strict mode overrides watermark → blocked
7
Request Access eligibility
Final gate. If user is blocked but the block is approvable (rights owner could grant a one-off pass), show Request Access. Not available for Internal Only assets or Expired assets — there's no human approver who can override these.
Block + recoverable → request
Block + not recoverable → blocked

⚠ Open questions surfaced by the matrix

  • U04 (UK Designer) + Editorial Only (Active) currently shows watermark — but this conflicts with the principle that Editorial blocks commercial use entirely. Should this be blocked? Resolution depends on whether internal designers ever have a legitimate reason to reference an editorial asset (e.g. as moodboard input). Recommend: watermark with explicit "reference only — not for use" overlay.
  • Royalty-Free + Public Share Link (Active) shows watermark as a tenant policy default — but the licence permits public delivery. Whether to watermark on share is a tenant policy choice, not a licence rule. Should this matrix represent the licence rule or the typical tenant config? Currently shows typical tenant config — flag for dev.
  • Rights-Managed + U06 (Agency Designer) (Embargoed) — RESOLVED. Cell now shows request. Agencies on the production team often need pre-launch access; approval routes to rights owner. If granted, access remains embargoed + watermarked until go-live (no exfiltration risk). Editorial and Internal Only embargoes still block agencies — those licence types have no agency entitlement at any point in the lifecycle.
  • Editorial Only + U09 (Content Editor) (Embargoed) — Press embargoes are a real use case (announcing a story under embargo to journalists). Current matrix blocks them. Worth a dedicated "embargo allows press" rule? Depends on whether press users carry an embargo-acknowledgement flag in their profile.
  • QA tester user model conflates audience and role — the existing QA tester (U01–U15) stores values like "Design", "Marketing", "Editorial", "Agency", "Broadcast" in an audiences array on each user. These are roles, not audiences. This matrix reclassifies them correctly (Audience: Internal / Agency / Press / Public; Role: Design / Marketing / Editorial), but the underlying QA test data still needs to be updated so the test plan and the rights engine evaluate against the right user attribute. Worth a focused QA-tester refactor pass.
  • USA Designer persona missing — every USA persona in the QA set has Marketing role. There's no USA + Internal audience + Design role persona to test "USA internal user with non-commercial intent" cases. U07 (USA Regional Manager, Marketing) currently stands in but produces Editorial Only blocks because of Marketing role — not because of audience. Adding a USA Designer persona would give cleaner test coverage.
  • Asset Owner download on Expired — RESOLVED. No direct download path. Owner must extend the licence (creating a new rights record) before any download is possible. This forces a fresh rights decision and prevents silent re-use of expired content. Trades convenience for safety. Triggers the licence creation/extension flow described below.
  • DAM Manager scope — currently modelled as full bypass across all tenant assets. Worth confirming this is correct vs. scoped bypass (e.g. only assets in collections they own). Full bypass is the simpler model and matches industry convention; scoped bypass is more secure but introduces complexity in collection-membership rules.
