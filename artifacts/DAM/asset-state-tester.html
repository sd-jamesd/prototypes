Production spec · canonical CSS

Card dot 10px · lead signal
Stands alone in the card detail row. Larger because colour is the only signal — no text label.
Chip dot 8px · supports label
Inside the chip on list rows and the lightbox. Smaller because the text label carries the meaning; the dot punctuates it.
Colours
Archived stone-600
In approval #3b82f6
On workflow purple-500
Canonical CSS
/* Shared treatment — colour, ring, shadow are identical; size adapts by role. */ .state-badge, .chip-dot { border-radius: 50%; box-shadow: 0 0 0 2px white, 0 0 0 3px rgba(0, 0, 0, 0.08); flex-shrink: 0; } .state-badge { width: 10px; height: 10px; cursor: pointer; position: relative; } .chip-dot { width: 8px; height: 8px; } /* Colours — apply to both contexts via parent class */ .state-badge.archived, .state-chip.archived .chip-dot { background: var(--color-stone-600); } .state-badge.approval, .state-chip.approval .chip-dot { background: #3b82f6; } .state-badge.workflow, .state-chip.workflow .chip-dot { background: var(--color-purple-500); }
Markup
// Card — dot alone in detail row <span class="state-badge archived" data-tooltip="Archived"></span> // Chip — dot + label in list rows and lightbox <span class="state-chip archived"> <span class="chip-dot"></span>Archived </span>
Rules
  • One axis = one colour. No multi-state encoding within a single dot.
  • Ring and shadow are constant across sizes. Only diameter changes.
  • Card dot stands alone (no label); chip dot punctuates a text label.
  • Hover tooltip required on every card dot. Label text inside chip serves the same purpose; no chip tooltip needed unless adding extra context.
  • Never invent a new colour for a new state without updating this spec first.
