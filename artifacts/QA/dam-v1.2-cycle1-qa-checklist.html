When something fails — how to log it

  1. Pass → tick the box. Done.
  2. Fail → in Linear, click + / New issue to create a fresh issue. (It's a new issue — not an action on the original.)
  3. Set the Bug label + a QA Severity label (Blocker / Major / Minor), assign the developer, cycle = Cycle 2.
  4. Add a "Relates to" link to the item's SCR-00 reference. Note what you did, expected, and saw. Leave the original issue untouched.
  5. Dev fixes → sets Ready for QA; you move to In QA, re-test, then Done.

Cross-cutting rules — check on every surface

A surface can pass its own cases and still break one of these. If one fails, log it against whichever item surfaced it.
